Big Bird helps you write more modular JavaScript.

View on Github

First of all, Big Bird is not a fully fledged framework like Backbone or Ember. It's designed for use when you don't need a MVC framework, but need something simpler and are fed up of writing monolithic jQuery based applications.

Core principles

  • Provide elegant sturcture to your JavaScript applications.
  • Each module should have one responsibility (SRP)
  • Modular JavaScript is easier to test in isolation
  • Modules should communicate via events using the pub / sub API
  • Don't do too much. If you need more, use a more fully featured framework

Initialiser

The intialiser allows you to structure your application into distinct sections that are tied to pages through the DOM. It's built ontop of DOM Ready Execution by Paul Irish.

This fits in really well with an MVC framework like Rails as your sections relate to controllers, and the actions to functions defined on the body of the application.

Module

The Big Bird module is a way of encapsulating class-like logic into one Javascript file. This means that a Module could represent anything from a like button, to a carousel.

A module has methods defined already that help you to perform some of the most common parts of JavaScript functionality like events, proxying functions and manipulating elements on the DOM.
